## 싱글 사이클 → 5단계 파이프라인 CPU 주요 수정 사항

### 1. **파이프라인 레지스터 추가** ⭐


// 4개의 파이프라인 레지스터 세트 추가

- IF/ID   (IFID_pc, IFID_inst)
- ID/EX   (IDEX_pc, IDEX_rs1_data, IDEX_rs2_data, IDEX_rd, 제어신호들...)
- EX/MEM  (EXMEM_aluout, EXMEM_rs2_data, EXMEM_rd, 플래그들, 제어신호들...)
- MEM/WB  (MEMWB_aluout, MEMWB_MemRData2RF, MEMWB_rd, 제어신호들...)


* 각 단계 사이에 레지스터를 삽입하여 데이터와 제어 신호 저장
* 매 클록마다 파이프라인 레지스터 업데이트

### 2. **Hazard Detection Unit 추가** 🛑


// Load-Use Hazard 감지
assign stall = (IDEX_MemtoReg &&
                ((IDEX_rd == rs1) || (IDEX_rd == rs2)) &&
                (IDEX_rd != 5'b0));



* Load 명령어 직후 해당 레지스터를 사용하는 경우 감지
* Stall 신호 생성 → 파이프라인 일시 정지

### 3. **Forwarding Unit 추가** ⚡




// EX-EX Forwarding (MEM stage → EX stage)
forwardA = (EXMEM_RegWrite && EXMEM_rd == IDEX_rs1) ? 2'b10 : ...

// MEM-EX Forwarding (WB stage → EX stage)
forwardA = (MEMWB_RegWrite && MEMWB_rd == IDEX_rs1) ? 2'b01 : ...

* Data Hazard를 forwarding으로 해결
* 2-bit 제어 신호로 forwarding 경로 선택

### 4. **Flush 메커니즘 추가** 💨


// Branch/Jump taken 시 잘못된 명령어 제거
assign flush = btaken | EXMEM_jal | EXMEM_jalr;

// IF/ID, ID/EX, EX/MEM 레지스터 초기화
if (~reset || flush) begin
  IFID_inst <= 32'h00000013;  // NOP
  // 모든 제어 신호 0으로 초기화
end

* Control Hazard 처리
* Branch/Jump taken 시 IF, ID, EX 단계 flush
* 3 사이클 penalty

### 성능 영향

**장점:**

* ✅ 처리량(Throughput) 향상: 이상적으로 5배 향상
* ✅ 여러 명령어가 동시에 다른 단계에서 실행
* ✅ CPI ≈ 1 (Hazard가 없는 경우)

**단점:**

* ❌ 레이턴시(Latency) 증가: 한 명령어는 5 사이클 필요
* ❌ Hazard로 인한 stall/flush: CPI > 1
* ❌ 하드웨어 복잡도 증가

**Hazard Penalty:**

* Load-Use: 1 사이클 stall
* Branch/Jump taken: 3 사이클 flush
* Branch not-taken: 0 사이클 (정상 진행)
